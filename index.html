Análise Orientada a Objetos (AOO)

A Análise Orientada a Objetos (AOO) é uma abordagem de desenvolvimento de software que modela um sistema como um grupo de objetos que interagem entre si, aproximando a estrutura computacional do mundo real.

1. Conceitos Fundamentais

  • Classe: Um "molde" ou blueprint que define atributos (características) e métodos (comportamentos) de objetos semelhantes.
  • Objeto: Uma instância concreta de uma classe. Ex: Carro (Classe) -> MeuFusca (Objeto).

2. Os 4 Pilares da POO/AOO

Encapsulamento: Esconder detalhes internos e proteger dados.
Herança: Reutilizar atributos/métodos de classes pai (superclasse) para classes filhas.
Polimorfismo: Objetos diferentes respondendo à mesma mensagem de formas distintas.
Abstração: Focar nos aspectos essenciais do objeto para o contexto atual.

3. Exemplo Prático (UML)

A Linguagem de Modelagem Unificada (UML) é usada para entender esses conceitos.

+-------------------------+

|      Funcionario        |  <-- Classe
+-------------------------+

| - nome: String          |  <-- Atributo (Encapsulado)
| - salario: double       |
+-------------------------+

| + trabalhar(): void     |  <-- Método
| + calcularBonus(): void |
+-------------------------+
